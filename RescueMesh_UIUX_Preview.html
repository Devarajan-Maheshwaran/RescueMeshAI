MVP Android UI/UX flow

This self-contained preview mirrors the implemented RescueMesh Android Views/XML navigation: emergency-first dashboard, one-tap SOS path, role-based coordinator view, trusted/offline status wording and priority feed cards.

Emergency-first interaction model

1. Dashboard
Critical SOS is the dominant action and is always reachable from RescueMesh navigation.
2. SOS composer
Sender selects urgency; English offline suggestion is visible but cannot override the sender.
3. Trusted offline queue
Message status uses factual language: stored, queued, synchronised to peer, acknowledged or expired.
4. Triage
Feed is priority ordered. Rescuer/Coordinator roles can acknowledge an individual SOS.

Presentation notes

  • No cloud account, Internet, Firebase or map dependency appears in the emergency path.
  • Location is explicitly optional and one-time.
  • The UI avoids stating “delivered to rescue service.”
  • Public mesh and guaranteed delivery are not represented as completed capabilities.
